# Симулятор айтишника

Браузерная idle-игра: один игрок, состояние в памяти (без БД). **Фронтенд:** React + Zustand (Vite), **бэкенд:** FastAPI с JSON API.

## Запуск для разработки

Терминал 1 — API (порт 8000):

```bash
python3 -m uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

Терминал 2 — Vite (порт 5173, прокси `/api` и `/audio` на 8000):

```bash
cd frontend && npm install && npm run dev
```

Открывай приложение по адресу Vite (обычно `http://127.0.0.1:5173`). Сборка фронта для продакшена не обязательна в деве.

## Продакшен (один процесс)

Собери фронт и отдавай всё через FastAPI:

```bash
cd frontend && npm install && npm run build
cd .. && python3 -m uvicorn main:app --host 0.0.0.0 --port 8000
```

Корень `http://127.0.0.1:8000/` отдаёт SPA из `frontend/dist/`, API: `GET /api/state`, `POST /api/work`, `POST /api/fun`, и т.д. Если `frontend/dist/` нет, `GET /` вернёт 503 с подсказкой собрать фронт.

## Docker

Образ собирает фронтенд внутри multi-stage Dockerfile и копирует `frontend/dist` в финальный образ.

```bash
docker compose build --no-cache && docker compose up
```

Открой в браузере один из адресов:

- **http://localhost:8000/** или **http://127.0.0.1:8000/** — основной вариант (порт проброшен как `8000:8000`).
- **http://localhost/** — если используешь проброс `80:8000` (без номера порта).

**Не открывай `http://0.0.0.0:8000/`** — в браузере это ненадёжно (часто пустая страница или ошибка). `0.0.0.0` в `uvicorn` — это «слушать на всех интерфейсах» внутри контейнера, а не адрес для вкладки браузера.

Если страница белая: в DevTools → **Сеть** проверь, что `GET /` отдаёт HTML, а запросы к **`/assets/*.js`** не **404** (иначе не подгрузился бандл).

## API

| Метод | Путь | Тело |
|-------|------|------|
| GET | `/api/state` | — |
| POST | `/api/work` | `{}` |
| POST | `/api/fun` | `{"activity": "<id>"}` |
| POST | `/api/buy_laptop` | `{"name": "..."}` |
| POST | `/api/apply_job` | `{"name": "..."}` |
| POST | `/api/buy_home` | `{"name": "..."}` |
| POST | `/api/buy_transport` | `{"name": "..."}` |

Ответы содержат полный снимок состояния (игрок, лог, каталоги, треки музыки).

## Механики

### Статы

| Стата | Диапазон / тип | Старт |
|-------|----------------|-------|
| Энергия | 0–100 | 100 |
| Настроение | 0–100 | 100 |
| Деньги | целое число | 100 |
| Опыт (XP) | целое число | 0 |
| Уровень | целое число | 1 |
| Работа | строка | «офисный шнырь» |
| Ноутбук | строка или нет | нет |

Энергия и настроение после действий ограничиваются диапазоном 0–100.

### Действия

- **Работать** — энергия −15, настроение −5, деньги: случайная сумма в диапазоне дохода текущей работы, +10 XP. Недоступно, если энергия меньше 15.
- **Развлечения** (раздел внизу): **Прогулка** — 0₽, настроение +5; **Выпить пива** — 200₽, настроение +25; **Сходить в сауну** — 5000₽, настроение +70. Платные варианты недоступны, если не хватает денег.

### Уровни

Для перехода на следующий уровень нужно **N×100 XP**, где **N** — текущий уровень. При повышении уровня лишний XP переносится, уровень +1, энергия сбрасывается до 100.

### Карьера и ноутбуки

Должности отличаются **минимальным уровнем**, иногда **требуемым ноутбуком** и **диапазоном дохода** при работе. Устроиться можно через кнопки в разделе «Работа», если выполнены уровень и ноутбук (если указан).

Ноутбуки покупаются за деньги; у игрока хранится один текущий ноутбук (покупка нового заменяет предыдущий).

### Лог

Внизу главного экрана — последние **5** сообщений о действиях и ошибках (недостаточно уровня, нужен ноутбук, не хватает денег и т.д.).

### Фоновая музыка

Файлы с расширением **`.mp3`** (регистр не важен: `.mp3`, `.MP3`) кладите в `audio/music/` — подхватываются по алфавиту и играются **по кругу**. Если один файл — он повторяется.

Браузеры часто блокируют автозапуск звука до клика: при необходимости появится кнопка «Включить музыку».

### Редиректы

- `GET /homes` → `/?tab=shop&store=cygan`
- `GET /transport` → `/?tab=shop&store=korita`

Параметры `?tab=`, `?store=`, `?worktab=` и устаревший `?open=homes|transport` обрабатываются на клиенте при загрузке.
